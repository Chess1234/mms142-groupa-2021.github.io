EDGE COMPUTING

"The principle is straightforward: If you can't get the data closer to the data center, get the data center closer to the data."

- Stephen Bigelow

WHAT IS EDGE COMPUTING

Edge Computing is an "alternative" way to compute, store, and process data. It is set in between the device and the cloud to help lessen latency and increase computing and analysis speed.

It enables the selective processing of data at different levels. Then, it sorts them into two main categories as follows.

  • “Hot” Data: Hot data provides a critical signal that must be transferred to the supervision system of the production line to act accordingly.
  • “Cold” Data: Cold data is typically a series of historical data containing parameters like pressure and temperature, which is helpful for subsequent predictive processing.

BENEFITS OF EDGE COMPUTING

  • Speed - Multiple localized micro data centers = less reliance on cloud data centers which avoids bandwidth traffic, latency issues, etc.
  • Security - Decentralized computing architecture makes it harder for attacks to be devastating (malware, phishing, DDOS) as the processes, storages, and applications are spread across multiple localized data centers.
  • Scalability - It is not demanding in terms of resources compared to having to maintain a fully-pledged data center that needs proper maintenance and substantial bandwidth allocation.
  • Versatility - By having edge data centers, it is possible to make changes depending on the end-users preference or target a market trend without completely altering the IT infrastructure.
  • Reliability - Edge computing chases almost all loose ends of a centralized data center. It reduces the risk of complete collapse of the server in case of an outage, data can be rerouted through multiple gateways that increases the speed of which you get the information and it increases efficiency of data exchange between two machines.

TYPES OF EDGE CLOUD

  • Fog computing puts data, compute, storage and applications closer to the user or Internet of Things device where data needs processing. This creates fog outside the centralized cloud and reduces the data transfer times necessary to process data.
  • Multi-access Edge Computing has ultra-low latency, high bandwidth and real-time access to radio network informations. According to the European Telecommunications Standards Institute (ETSI) definition, this offers application developers and content providers cloud-computing capabilities and an IT service environment at the edge of the network.
  • Micro data centers are highly mobile and rugged. When highly flexible, they can be custom built and configured to suit the implementation requirements of unique situations. This allows them to be rapidly deployed to underserved areas or something like disaster centers.
  • Cloudlets are modeled after clouds. They are mobile small-scale data centers placed in close proximity to edge devices. They improve resource-intensive and interactive mobile apps through low-latency computing resources.
  • Emergency response units are self-contained and they establish interoperable communications for first responders in emergency situations. It makes them easy to deploy to any crisis site with the help of a highly skilled Tactical Operations team.
